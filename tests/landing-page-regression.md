# Landing-page regression scenario

This scenario is the primary behavioral RED test for Product0 v0.2.

## User request

```text
Use Product0 in the running repository:
we need to design a simple landing page for the app.
```

## Repository conditions

Assume the repository contains some combination of:

- README or product overview;
- existing product and architecture docs;
- prior design/spec artifacts;
- public app routes and copy;
- screenshots or visual assets;
- pricing or demo-phase notes;
- recent commits.

## Required behavior

1. Inspect relevant repository evidence before product questions.
2. Present a synthesized `Lay of the land` with repository-relative references.
3. Explain what the page should probably accomplish and why.
4. Create no Product0 brief or placeholder directories during orientation/discovery.
5. Focus on audience, buying situation, page job, positioning, message hierarchy, differentiation, proof, objections, product demonstration, visual direction, conversion, measures, and claim safety.
6. Use no more than two decision packets for an appropriately small landing page unless a genuine blocker appears.
7. Make low-risk choices when the user delegates judgment.
8. Preserve “likely 1%” as provisional.
9. Challenge the proposed direction before user approval.
10. Write one concise brief only after direction approval.
11. Include relevant repository evidence and provenance.
12. Omit requirement IDs, lifecycle states, or product slices when they add no value.
13. Stop at developer handoff.

## Forbidden behavior

- Creating a brief before discovery.
- Updating the brief after every answer.
- Opening with a basic question the repository can answer.
- Asking routine validation, duplicate, retry, loading, or accessibility questions before strategic direction is established.
- Treating a generic product checklist as the conversation.
- Asking the user to approve a delegated low-risk choice.
- Converting provisional commercial language into a fixed claim.
- Producing a repetitive, template-complete document with no evidence.
